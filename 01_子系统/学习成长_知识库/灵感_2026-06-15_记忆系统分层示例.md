# 示例：记忆系统也许不该是单一向量库，而是…

> 这是一篇**示例**知识库笔记，展示沉淀后的结构（一句话核心 / 关键点 / 今日意义 / 相关 `[[双链]]`）。
> 真实笔记由「沉淀知识」/「归档」按钮调用 LLM 自动生成，模仿这个结构写就行。

### 1) 一句话核心
分层记忆（短期滑动窗口 → 中期摘要压缩 → 长期向量库）比单一向量库更抗噪声、更省算力，但分层本身不是银弹——边界模糊和摘要幻觉是真正的坑。

### 2) 关键点（带可复用的接口/方法/结论）
- **滑动窗口动态调整**：固定窗口大小可能浪费或遗漏，建议窗口大小 = "最近 N 轮对话" + "当轮 Token 数"的加权组合，并在语义转折点提前截断。
- **中期摘要主动纠偏**：每次对话结束后生成摘要，强制附加"置信度评分"和"原始引用位置"，矛盾时用置信度加权投票修正。
- **长期入库触发条件**：不要每次都入库——只有摘要累积变化超阈值或涉及全新主题时才触发，避免高频写入噪声。
- **转化门限（盲点解法）**：中期摘要不应直接入长期库，应过一道"鲁棒性过滤层"——用另一个 LLM 检查摘要与原文一致性，不一致则退回重生。

### 3) 今日意义
- **研究/工作上**：若你在搭多轮对话系统，这给了可直接落地的实验框架——先切短期（最近对话列表）和长期（向量库），中期摘要用简单规则试水。
- **学习方法上**：个人知识管理也可分层——碎片笔记（短期）、每周总结（中期）、领域知识图谱（长期）。每篇笔记加"置信度"和"来源引用"字段，防后期失真。
