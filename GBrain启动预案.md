# GBrain 启动预案

> 给未来的你看的。现在**不要装** GBrain——它是给"知识已海量、要交给 Agent 自主管"的阶段用的。
> 本系统现在刻意保持「纯 Python 标准库、零依赖」。这份文档只是把"将来要用时怎么接"提前写好，
> 让你到点能一键启动，不用重新研究。

---

## 一、为什么现在不装（先读这段，别冲动）

- 装着不用 ≠ 零成本：依赖会过期、版本会烂，等真要用往往还得重装。
- GBrain 要 Bun + Postgres，一装，本项目就**失去"有 Python 就能跑"**这个最大优点。
- GBrain 的"做梦周期"（几十个 cron 自动改你的知识）恰是总纲领警告的反模式：
  "记录花的时间不能超过它省的时间" + "别让 AI 自动写入污染记忆核心"。

**判断口诀：不是"项目大了就要上 GBrain"，而是"flat 文件真的搜不动了才上"。**

---

## 二、什么时候才启动（满足任一条，且确认不是一时冲动）

1. **规模**：`学习成长_知识库/` > ~300-500 篇（看板会在达 300 时自动弹"绊线"提醒）。
2. **痛感**：开始反复出现"我明明存过这个，却找不到 / 翻半天找不到"。
3. **需求升级**：你想要的从"列出相关文件"变成"直接给我带引用的答案 + 告诉我还缺什么"
   （即 GBrain 的 `think` + 缺口分析），而手工翻已经满足不了。

> 三条都不满足 = 还不到时候，继续用现在这套。**晚启动的代价远小于早启动。**

---

## 三、我们已经为它预埋了什么（所以将来很省事）

- **数据格式天然兼容**：GBrain 的"系统记录"本身就是 git 里的 markdown，它只是把 md 同步进 Postgres 做检索。
  我们的复盘/规则/知识库**已经都是 markdown**，无需转换。
- **`[[双链]]` 已在用**：`learn`/`pit` 沉淀时 AI 已按约定在文末标 `相关：[[R1]] [[2026-06-15]]`。
  GBrain 的图谱抽取器读的正是这个语法 —— 启动后这些链**直接变成知识图谱的边，零返工**。

## 四、启动步骤（到点了照着做，约 30 分钟）

> 项目地址：https://github.com/garrytan/gbrain（MIT）。以下命令以官方 README 为准，执行前先扫一眼最新文档。

### 1. 装运行时
```bash
# 装 Bun（GBrain 的运行时）
# Windows: powershell -c "irm bun.sh/install.ps1 | iex"
bun install -g github:garrytan/gbrain
gbrain doctor          # 自检环境
```

### 2. 建大脑（用零配置的 PGLite，不用单独起 Postgres）
```bash
cd "<你的本项目根目录>"
gbrain init --pglite   # PGLite = WASM 版 Postgres，2 秒就绪，撑到 ~5 万页
```

### 3. 把整个项目喂进去
```bash
gbrain import .        # 吃掉所有 md：总纲领/子系统/规则库/复盘/知识库
                       # [[双链]] 会自动抽成知识图谱的边
```

### 4. 配 LLM
- 你本系统用的 LLM（在 `tools/models.json` / `.env` 里）若 GBrain 默认 recipe 没收录，有两条路：
  - **省事**：用 GBrain 自带某家的 embedding（如 OpenAI / Ollama 本地），只把"检索"交给它；
  - **统一**：照 GBrain 的 recipe 格式，给你的提供商写一个 OpenAI 兼容 recipe（多数国产兼容接口本就是 OpenAI 兼容）。
- ⚠️ 复用本项目密钥时，**别把密钥写进任何会进 git 的文件**（.env / models.json 已被 gitignore，保持这样）。

### 5. （可选）接进 Claude Code
```bash
claude mcp add gbrain -- gbrain serve   # 本地 stdio，单机单用户够用
```

### 6. 验证
```bash
gbrain search "22:00 规则"     # 看能不能检索到 R1
gbrain think "我的系统现在缺哪类规则？"   # 综合 + 缺口分析
```

---

## 五、启动后，看板怎么办

- 现在的 `tools/server.py` 看板**继续用**——它读的还是同一批 md 文件，不冲突。
- 可选增强：给看板加个"问知识库"输入框 → 后端调 `gbrain think` → 返回带引用的答案 + 缺口。
  那时才真正用上 GBrain 的核心价值（综合而非罗列）。

---

## 六、退出 / 回退

GBrain 不动你的 md 源文件（git 是系统记录，DB 只是它的检索副本）。不想用了：
- 停掉 `gbrain serve` / 删掉 MCP 配置即可，**md 文件原封不动**，本项目退回纯 Python 看板。
- 这就是为什么"先就绪、不集成"安全：**随时能上，随时能撤，数据始终在你手里。**
