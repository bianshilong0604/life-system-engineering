# 系统说明

> 这套「个人总体设计部」是什么、怎么运转。第一次用先读这个。

## 一句话
把"自我管理"做成一个**可运行、可反馈、可进化**的小系统，灵感来自钱学森系统工程。每周 15 分钟复盘，系统自己变强。

## 核心闭环（钱学森方法论）
```
Plan → Execute → Verify → Reflect → Patch
```
- **Plan**：每周给每个活跃子系统定 1-2 件（少而精）
- **Execute**：这周做
- **Verify**：周末看真实完成度（不看"看起来很忙"，看具体推进了什么）
- **Reflect**：没做到的——能力问题还是系统问题？系统问题再归类
- **Patch**：系统问题提炼一条规则进规则库，下次自动减少

## 目录结构
```
00_总纲领.md          ← 总纲：硬约束、子系统、三个核心问题
01_子系统/            ← 3 个活跃子系统（研究工作/学习成长/复盘进化）
  学习成长_知识库/    ← 沉淀的知识笔记
02_每周复盘/          ← 每周复盘记录（YYYY-MM-DD.md）
03_今日见闻/          ← 今日见闻池（YYYY-MM-DD.json）
规则库.md             ← 提炼出的可执行规则（R 编号）
规则库_候选.md        ← 新规则先放这里，复盘确认后才搬进规则库
AI复盘助手.md         ← 给 Claude Code 等 LLM 工具的复盘 Prompt
LLM配置说明.md        ← 怎么配 LLM API（密钥/模型/温度）
GBrain启动预案.md     ← 将来知识库大了再看的进阶方案（现在不用）
tools/                ← 纯 Python 标准库，零依赖
  assistant.py        ← CLI：复盘/踩坑/沉淀知识/月度体检
  server.py           ← 可视化看板（本地 web）
  jianwen.py          ← 今日见闻池：入池/归档/丢弃
  prompts.py          ← 所有 LLM Prompt 唯一来源
  config.env.example  ← LLM 配置模板（复制成 .env 填密钥）
  .env / models.json  ← 你的真实密钥（已 gitignore，不外传）
```

## 两种用法

### A. 纯手动
每周日晚打开 `02_每周复盘/` 新建一篇，照着四段式手写。最低维护成本。

### B. AI 辅助
- **命令行**：`python tools/assistant.py review`（带配置好密钥的 .env，见 LLM配置说明.md）
- **Claude Code**：打开 [AI复盘助手.md](AI复盘助手.md)，把 Prompt A 复制给它，它读你的文件、带你跑闭环、自动写回复盘
- **可视化看板**：`python tools/server.py`，浏览器里看/改一切，点按钮触发 AI 复盘

## 防过度系统化（最重要的原则）
- 指标少到**一只手数得过来**，记录花的时间不能超过它省的时间
- 关系、创造力、当下的快乐——**不该被量化**
- **如果这套东西让你更累更焦虑，就是搭错了，立刻简化**
- 系统是工具，不是目标
