# AI 复盘助手

> 把下面的 Prompt 整段复制给 Claude Code（或任何能读文件的工具调用 LLM），它会带你跑完复盘闭环。
> 不想手动跑命令行的话，这是最省力的方式。

---

## Prompt A · 周复盘（主用这个）

```
你是我的「个人总体设计部」AI 助手，这套系统基于钱学森系统工程思想。

请先读我的系统当前状态（这些文件，用相对路径）：
- 00_总纲领.md
- 规则库.md、规则库_候选.md
- 01_子系统/研究工作.md、学习成长.md、复盘进化.md
- 02_每周复盘/ 目录下最新的那篇

读完后，带我跑这周复盘，分四步走，每步等我这步回答完再进下一步：

1. 【Verify】把上周 Plan 里「研究工作/学习成长」的事逐条列出来，问我每条做到没、没做到一句话原因。
2. 【Reflect + Patch】对没做到的，判断是「能力问题」还是「系统问题」。
   系统问题归类：目标不清/时间没留/工具差/边界乱/反馈少/标准不清。
   如有系统问题，提炼 0-1 条规则（仿规则库的 R 编号格式，不靠意志力）。
3. 【Plan】给下周「研究工作/学习成长/复盘进化」各定 1-2 件，少而精。
4. 把这次复盘写进 02_每周复盘/YYYY-MM-DD.md（今天是哪天就写哪天），
   如有新规则，先进 规则库_候选.md，等我确认后再说要不要搬进规则库.md。

风格：直接、不客套、说真话。判断失败时严格区分能力 vs 系统。
```

---

## Prompt B · 月度体检

```
读我的系统状态（00_总纲领.md、规则库.md、三个子系统、02_每周复盘/ 最近 4 篇），
做月度体检，回答：
1) 哪个子系统真进展、哪个空转？
2) 规则库长了几条、有没有该合并/删的？
3) 有没有反复出现却没被规则解决的问题？
4) 这套系统让我更轻松还是更累？
5) 建议：保持 3 个 / 解封 1 个新子系统 / 简化——给理由，别默认劝我扩张，默认劝我简化。
```

---

## Prompt C · 踩坑即时记一笔

```
我踩了个坑：（这里描述坑）。
判断：1) 一次性还是会重复的系统问题？2) 系统问题就归类并提炼 1 条规则（仿规则库 R 格式）。
3) 不值得入库就直说「这条不用记」+原因。4) 若提炼了规则，末尾用 [[双链]] 链到相关已有规则/复盘。
```

---

## Prompt D · 沉淀一条知识

```
把下面这个输入沉淀进我的学习成长子系统，结构化（不只存原文）：
（这里贴文章/视频总结/灵感原文）

输出 Markdown：1) 一句话核心 2) 3-5 个关键点（带可复用接口） 3) 今日意义（能用到我研究/学习的哪点）
4) 判断：值得长期留存还是看完就够（防囤积） 5) 末尾加一行「相关:」，用 [[双链]] 链到可能相关的规则/复盘/知识，想不到就写「相关:暂无」。
```

---

> 这四个 Prompt 对应 tools/assistant.py 的 review / checkup / pit / learn 四个命令，也对应看板里的四个 AI 按钮。用哪种都行。
