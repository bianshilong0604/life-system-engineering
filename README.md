# 个人总体设计部 · Life System Engineering

> 把"自我管理"做成一个**可运行、可反馈、可进化**的小系统。
> 灵感来自钱学森《论系统工程》——把"组织管理"上升为科学方法。
> 每周 15 分钟复盘，让系统自己变强。

---

## 这是什么

一套基于 markdown 文件 + 纯 Python 脚本的"个人系统工程"框架。它不是待办清单，而是一个会自我改进的闭环：

```
Plan（定）→ Execute（做）→ Verify（核对）→ Reflect（归因）→ Patch（提炼规则）
```

- **不靠意志力，靠规则**：把容易栽的坑提炼成可执行规则，下次自动减少。
- **少而精**：当前只聚焦 3 个子系统（研究工作 / 学习成长 / 复盘进化），其余封存。
- **AI 辅助，人定夺**：LLM 带你跑复盘、沉淀知识，但规则库的最终决定权始终在人手里。
- **零依赖、低维护**：纯 Python 标准库，有 Python 就能跑；平时不用管，只在周日复盘时打开。

## 谁适合用

- 觉得"自我管理工具"总是用着用着就放弃的人
- 想把碎片学习/灵感真正沉淀下来、而不只是收藏的人
- 一个人推进一个主线项目（研究/写作/创业）、需要守住精力边界的人
- 相信"系统比意志力可靠"的人

## 30 秒上手

```bash
# 1. 克隆
git clone <your-fork-url> life-system
cd life-system

# 2.（可选，想用 AI 功能才需要）配 LLM
cd tools/
cp config.env.example .env
# 编辑 .env 填你的 API Key（任何 OpenAI 兼容提供商都行）
python assistant.py test   # 测连通

# 3.（可选）启动可视化看板
python server.py   # 自动开浏览器
```

不想用 AI？跳过第 2 步，纯手动用——每周打开 `02_每周复盘/` 新建一篇，照着四段式手写即可。

详细见 [系统说明](系统说明.md) 和 [LLM配置说明](LLM配置说明.md)。

## 核心理念（摘自总纲领）

**三个核心问题（每次复盘必问）：**
1. 这次搞砸是**能力问题**还是**系统问题**？
2. 系统问题归类：目标不清 / 时间没留 / 工具差 / 边界乱 / 反馈少 / 标准不清？
3. 有没有**一条规则**能让同类问题下次自动减少？

**防过度系统化（最重要）：**
- 指标少到一只手数得过来，记录花的时间不能超过它省的时间
- 关系、创造力、当下的快乐——不该被量化
- **如果这套东西让你更累更焦虑，就是搭错了，立刻简化**
